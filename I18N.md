Visiting `/` redirects automatically based on the browser language.

Everything they contained now lives under `app/[lang]/`.

## 3. How it works

- **middleware.ts** ‚Äî any URL without a locale prefix is redirected;
  Arabic-language browsers get `/ar`, everyone else `/en`.
- **app/[lang]/layout.tsx** ‚Äî awaits `params.lang`, validates it, then renders
  `<html lang={lang} dir={lang === "ar" ? "rtl" : "ltr"}>`. Arabic pages use the
  Cairo font (`font-arabic`), English uses Inter. `generateStaticParams`
  pre-renders both locales; `generateMetadata` emits localized SEO tags and
  `hreflang` alternates.
- **Dictionaries** ‚Äî server-side only. Pages call
  `const dict = await getDictionary(lang)` and read strings from it. To add a
  language later: add `dictionaries/xx.json`, register it in
  `lib/dictionaries.ts`, add it to `locales` in `middleware.ts`.
- **Language switcher** ‚Äî the globe button in the navbar swaps `/en` ‚Üî `/ar`
  while keeping the current page.
- **RTL details** ‚Äî Tailwind logical properties (`ms-`, `start-`) and
  `rtl:rotate-180` on arrow icons make the layout mirror automatically. Phone
  numbers are wrapped in `dir="ltr"` so they read correctly in Arabic.

## 4. Still English-only (next step)

`translation`, `about`, and `contact` pages render English content in both
locales for now. To translate them, add matching sections to both dictionary
files and swap the hard-coded strings for `dict.*` references ‚Äî exactly the
pattern used in `app/[lang]/page.tsx` and `app/[lang]/academy/page.tsx`.

## 5. Logo usage

- Favicon: `app/icon.svg` ‚Äî Next.js serves it automatically, nothing to configure.
- Navbar/footer already use `/logo-mark.svg`.
- `public/logo.svg` is the full horizontal logo (mark + bilingual wordmark) for
  print, social covers, or the site itself.
